- Added `[global::System.CodeDom.Compiler.GeneratedCode]` attribute to all three generated type declarations (root type, step structs, existing partial types)
- Updated `AggressiveInlining` attribute to use `global::` prefix

## Decisions Made

- Used `SymbolDisplayFormat.FullyQualifiedFormat` with `UseSpecialTypes` option to get both `global::` prefix and C# keyword aliases (int, string, bool) -- this is more correct than manually handling special types
- Removed namespace parameter threading entirely rather than keeping it for backward compatibility -- global:: makes it unnecessary and the simpler API is better
- For `RegularFluentStep.IdentifierDisplayString()` (parameterless overload), used `Name` only since step structs reference themselves within the same namespace; the overload with `genericTypeArgumentMap` uses `ToGlobalDisplayString()` since mapped types may be concrete cross-namespace types
- Used `ParseTypeName` instead of `IdentifierName` at all call sites where global:: strings are produced, since `IdentifierName` treats the entire `global::System.String` as a single token
