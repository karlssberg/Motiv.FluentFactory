<objective>
Widen `[FluentCollectionMethod]` to full property parity: the attribute may be applied to collection-typed properties, producing the same `AccumulatorFluentStep` shape (AddX + WithXs, `ImmutableArray<T>` backing, free composition) with terminal-time assignment routed through the existing `TargetTypeObjectCreationExpression.PropertyInitializers` path. Introduce CVJG0053 for unsupported property accessor shapes (primary case: record primary-constructor positional property). Reuse CVJG0050 (existing parameter diagnostic for non-collection) for property-target non-collection misuse — no new descriptor needed for that case.

Purpose: Delivers the second half of COMP-01 coverage (combined attributes on a property) plus the property-parity scope lift locked in 23-CONTEXT.md. Required properties and init-only properties work via the v2.0 object-initializer path already emitted for properties; this plan reuses it rather than building parallel infrastructure.

Output: Widened attribute, new CollectionPropertyInfo, extended analyzer (parameter + property entry points), new CVJG0053 descriptor and release entry, extended model-builder + emitter routing, two replaced test-stub suites.
</objective>
