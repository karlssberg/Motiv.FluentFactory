Purpose: The fluent builder pattern stores values in struct fields, so parameter modifier semantics (ref, out, etc.) are lost. The generator should emit a diagnostic and skip such constructors entirely rather than silently dropping the modifier.
Output: New diagnostic descriptor, constructor filtering logic, and test file with edge case scenarios.
</objective>

<!-- Key detection point: IParameterSymbol.RefKind -->
Roslyn's IParameterSymbol has RefKind property:
- RefKind.None = normal parameter
- RefKind.Ref = ref parameter
- RefKind.Out = out parameter
- RefKind.In = in parameter (C# in keyword, not the generator's `in` usage)
- RefKind.RefReadOnlyParameter = ref readonly parameter
