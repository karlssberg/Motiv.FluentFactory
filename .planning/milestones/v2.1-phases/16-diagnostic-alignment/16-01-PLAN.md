<vocabulary_policy>
Apply per-descriptor with judgment. Every occurrence of "Factory", "factory", "fluent factory" must be classified before editing:

1. **"Factory" referring to the fluent root → `FluentRoot`** (attribute-cased).
   Examples:
   - "Factory type missing partial modifier" → "FluentRoot type missing partial modifier"
   - "Factory accessibility exceeds target type" → "FluentRoot accessibility exceeds target type"
   - "Inaccessible parameter type in fluent builder" — already correct, leave it
   - "static factory type" (XML doc) → "static FluentRoot type"

2. **Ambiguous tool/generator phrasings → "Converj generator"**.
   Examples:
   - "Diagnostic descriptors for the fluent factory source generator" → "Diagnostic descriptors for the Converj source generator"
   - "Fluent factory generation requires value-type parameters" → "Converj generator requires value-type parameters"
   - "cannot be used by the fluent factory" → "cannot be used by the Converj generator"

3. **"Factory" referring to C# constructors or the retained GoF pattern → UNTOUCHED**.
   Read context before replacing. If a title/message describes a C# constructor, leave it. If it describes the GoF-pattern builder helper, leave it.

4. **"fluent constructor" referring to a C# constructor that is a fluent target → UNTOUCHED**.
   This is descriptive English, not drift. E.g., "Unreachable fluent constructor" describes an unreachable C# ctor — leave it.

5. **CVJG0031 defect fix (line 367)**: Replace `[FluentFactory]` with `[FluentRoot]`. The attribute was renamed in v2.0; this message was missed during the v2.0 rename. Document in commit message as "defect discovered during Phase 16 vocabulary alignment."
