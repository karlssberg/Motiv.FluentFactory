<objective>
Rename the metadata trio — `FluentFactoryMetadata`, `FluentFactoryMetadataReader`, and `FluentFactoryDefaults` — to `FluentRootMetadata`, `FluentRootMetadataReader`, and `FluentRootDefaults`. Move all three source files via `git mv` so `git log --follow` preserves history. Update every call site inside `src/Converj.Generator/` to use the new type names. Closes NAME-03 and the FILE-01 obligation for these three files.

The public method names `GetFluentFactoryMetadata` and `GetFluentFactoryDefaults` on the renamed `FluentRootMetadataReader` class are **intentionally preserved** in this plan. Rationale: those method names contain the legacy "FluentFactory" vocabulary, but renaming them simultaneously with the type would force a wider edit blast (every consumer site would change in two dimensions at once). The method-name cleanup is a small, mechanical follow-up that can ride along with Phase 18 (which already renames `FluentModelFactory.CreateFluentFactoryCompilationUnit` for the same reason). This deferral is logged in the plan summary so Phase 18 picks it up.

Purpose: The metadata reader sits at the boundary between Roslyn attribute parsing and the generator's domain model. Aligning its type names with `[FluentRoot]` makes the data flow legible: read the `[FluentRoot]` attribute → produce `FluentRootDefaults` and `FluentRootMetadata`.

Output: Three renamed files (history preserved), updated call sites in three consumer files, green build, green test suite.
</objective>
